ARCHITECTURE
============

Simulator Daemon (Python http.server)
         ↓
         ├─ /metrics (current tick)
         ├─ /summary (aggregated)
         ├─ /export/json (full history)
         ├─ /export/xml (full history)
         └─ /preset/{name} (control)
         
         ↓ HTTP Requests
         
Simulator Bridge (SimulatorClient + SimulatorEventBridge)
         ↓ Converts metrics to event format
         ↓
Backend API (/api/simulator/*)
         ↓
Ingestion Pipeline → Event → Normalization → Analysis
         ↓
Health Reports & CI Evaluation

DATA FLOW
=========

Daemon (every 1 second):
  1. Advance subsystem state (CPU, RAM, users)
  2. Store in metrics_history (last 3600 ticks)
  3. Export to metrics_latest.json/xml every 5s

Backend (on demand):
  1. GET /api/simulator/metrics → fetch from daemon
  2. Convert to event format (SimulatorClient.metrics_to_event)
  3. POST to /api/events for ingestion
  4. Normalize and analyze

BACKEND INTEGRATION POINTS
==========================

1. SimulatorClient (Low-level):
   from src.simulator_bridge import SimulatorClient
   client = SimulatorClient("http://localhost:9999")
   metrics = client.get_current_metrics()
   event = client.metrics_to_event("system-name", "ci")

2. SimulatorEventBridge (High-level):
   from src.simulator_bridge import SimulatorEventBridge
   bridge = SimulatorEventBridge()
   event = bridge.pull_metric_event("system-name", "ci")
   
3. Quick Function:
   from src.simulator_bridge import pull_simulator_metric
   event = pull_simulator_metric("system-name", "ci")

4. Continuous Monitoring:
   bridge = SimulatorEventBridge()
   bridge.continuous_monitoring("system", "ci", interval=1.0, callback=process_event)

API ENDPOINTS
=============

Health & Status:
  GET /api/simulator/health              Check if daemon is running
  GET /api/simulator/metrics             Get current tick metrics
  GET /api/simulator/summary             Get aggregated statistics

Control:
  POST /api/simulator/preset/{preset}    Switch load pattern
  Valid: default, high_cpu, low_cpu, high_ram, low_ram, high_users, low_users

Ingestion:
  POST /api/simulator/ingest?system_name=X&environment=Y
    → Pulls metric, converts to event, ingests
    → Returns: {event_id, status, preset}

Export:
  GET /api/simulator/export/json         Full metrics as SQL-ready JSON
  GET /api/simulator/export/xml          Full metrics as SQL-ready XML

TROUBLESHOOTING
===============

Problem: "Simulator daemon not available"
  Solution: Start daemon - python3 infrastructure_simulator_daemon.py --port 9999

Problem: Backend cannot resolve simulator
  Solution: Check port 9999 is accessible: curl http://localhost:9999/health

Problem: High CPU during continuous ingestion
  Solution: Increase interval parameter (default 1.0s)
  Example: continuous_monitoring("sys", "ci", interval=5.0)

Problem: Memory grows when ingesting many events
  Solution: Backend should be batching to database
  Check: Are events being persisted and pruned from memory?

Problem: Metrics don't reflect preset change
  Solution: 1. Verify preset change: GET /api/simulator/metrics
           2. Wait 1-2 seconds for state to update
           3. Check daemon logs for errors
